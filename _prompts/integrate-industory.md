# Integrate: Industry v2.0

## プロンプトの目的

このプロンプトは、**Five Forces調査とCluster Analysis（Diamond Framework）調査の結果を接続し、業界構造の全体像を整理**することを目的とする。

Five Forcesが「What（競争構造はどうなっているか）」を記述するのに対し、ダイアモンドフレームワークは「Why（なぜその構造になっているか）」を説明する。

### ⚠️ 最重要原則：関係の整理に専念せよ

```
このプロンプトでやること：
✅ Five Forcesで観察された事実とダイアモンドの6要因を接続する
✅ 「この構造はこの要因によって生じている」という関係を整理する
✅ 構造の安定性/変化可能性に関する事実を整理する

このプロンプトでやらないこと：
❌ 「業界の魅力度が高い/低い」と評価しない
❌ 「参入障壁が強い/弱い」と判断しない
❌ Tier判定やg∞に言及しない
❌ 投資判断に関する示唆を述べない

評価・判断はDebateフェーズで行う。
ここでは「事実と事実の関係」を整理することに専念する。
```

## 入力

- Five Forces調査レポート
- Cluster Analysis調査レポート（Diamond Framework）

---

## ダイアモンドフレームワークの構造（再掲）

```
【4つの内生要因】
1. 要素条件（Factor Conditions）
2. 需要条件（Demand Conditions）
3. 関連・支援産業（Related and Supporting Industries）
4. 企業戦略・構造・競争（Firm Strategy, Structure, and Rivalry）

【2つの外生要因】
5. 政府（Government）
6. 偶発事象（Chance）
```

## 注意事項

### ❌ やってはいけないこと

```
❌ 「参入障壁は高く、持続性がある」と結論づける
   → ✅ 「要素条件（専門人材の希少性）、政府（認証制度）、
         関連・支援産業（サプライヤーとの長期関係）という
         3つの要因が新規参入の少なさと対応している」
         と関係を整理する

❌ 「ダイアモンドが強固だから競争力がある」と評価する
   → ✅ 「6要因のうち4要因がFive Forcesの安定に寄与している。
         2要因（政府、偶発事象）には変化の兆候がある」
         と事実を整理する

❌ 「この構造は5年後も維持される」と予測する
   → ✅ 「過去の類似事例では、政府の政策変更から
         構造変化まで3-5年を要した」と過去の事実を記録する
```

### ✅ 心がけること

```
✅ Five Forces（What）とダイアモンド（Why）の関係を明確にする
✅ 6つのダイアモンド要因を漏れなく検討する
✅ 要因間の相互作用（フィードバックループ）を特定する
✅ 説明できない事実は正直に「説明できない」と記録する
✅ 因果関係の強さ（信頼性）を明記する
```
