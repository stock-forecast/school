# Estimate: Beta v1.0

## プロンプトの目的

このプロンプトは、**株価変動ではなく事業基盤の変動性（ROEボラティリティ）に基づき、個別銘柄のベータを推定**することを目的とする。

### 設計思想

```
従来のベータ：株価の変動性（市場との共変動）から算出
　→ 短期的なノイズ、流動性、投資家心理の影響を受ける

本プロンプトのベータ：事業基盤の変動性から算出
　→ 企業の本質的なリスク特性を反映
　→ 株価データが不十分な企業にも適用可能
```

### ベンチマーク

| ベンチマーク | β | ROEボラティリティ（目安） |
|-------------|---|-------------------------|
| TOPIX | 1.0 | 基準 |
| J-REIT | 0.7 | 低（安定的な賃料収入） |

## 注意事項

### ❌ やってはいけないこと

```
❌ 株価ベータをそのまま使用する
   → ✅ ROEボラティリティから算出する

❌ 調整係数を根拠なく設定する
   → ✅ 入力レポートの事実に基づいて評価する

❌ 極端なβ（0.3未満、2.5超）を無検証で採用する
   → ✅ 妥当性チェックを実施し、必要に応じて再検討する

❌ 財務レバレッジの影響を二重計上する
   → ✅ ROEボラには財務レバレッジが含まれるため、
         調整係数は「業界平均からの乖離」で評価する
```

### ✅ 心がけること

```
✅ ROEボラティリティを出発点に客観性を担保する
✅ 5つの調整要因で定性面を反映する
✅ J-REITをベンチマークとして妥当性を検証する
✅ 調整の根拠を明確に説明する
✅ 推定の限界を正直に記載する
```

## 参考：調整係数の設計根拠

### 調整係数の合計範囲

```
5要因 × 各±0.10〜0.15 = 最大±0.60程度

想定されるβの範囲：
- 基準β 0.8 + 調整-0.3 = 0.5（ディフェンシブ）
- 基準β 1.0 + 調整±0.0 = 1.0（市場平均）
- 基準β 1.2 + 調整+0.3 = 1.5（景気敏感）
```

### 財務レバレッジの二重計上回避

```
ROEボラティリティには財務レバレッジの影響が含まれる：
ROE = ROA × 財務レバレッジ

したがって、財務レバレッジの調整係数は
「業界平均からの乖離」のみを反映する：
- 業界平均並み → 調整なし（ROEボラに反映済み）
- 業界平均より高い → 追加調整（+方向）
- 業界平均より低い → 追加調整（-方向）
```
