# Research: Value Chain v1.0

## プロンプトの目的

このプロンプトは、**企業の事業単位における価値創造の仕組みを事実ベースで収集**することを目的とする。

### ⚠️ 最重要原則：事実収集に専念せよ

```
このプロンプトでやること：
✅ 主活動・支援活動の具体的な内容を収集する
✅ 各活動のコスト構造、効率性を定量化する
✅ 競合との比較データを収集する
✅ 活動間の連携（リンケージ）を記述する

このプロンプトでやらないこと：
❌ 「競争優位がある/ない」と評価しない
❌ 「強み/弱み」と判断しない
❌ Tier判定やフェードレートに言及しない
❌ 投資判断に関する示唆を述べない

評価・判断はDebateフェーズで行う。
ここでは「根拠となる事実」を揃えることに専念する。
```

---

## 入力

- 企業名
- 分析対象事業（複数事業の場合は特定）
- 主要競合企業名（比較用）

## 品質チェックリスト

収集完了前に以下を確認：

### 網羅性
- [ ] 主活動5つすべてについて情報を収集したか
- [ ] 支援活動4つすべてについて情報を収集したか
- [ ] 活動間リンケージを特定したか

### 定量性
- [ ] 各活動のコスト比率を把握したか
- [ ] 効率性指標（生産性、回転率等）を収集したか
- [ ] 時系列での変化を追跡したか

### 客観性
- [ ] 事実と意見を分離したか
- [ ] 「強い」「弱い」「優れている」等の評価表現を避けたか
- [ ] 競合比較は数値ベースで行ったか

### 追跡可能性
- [ ] すべての情報に出典を付けたか
- [ ] 情報源の信頼性を分類したか

## 注意事項

### ❌ やってはいけないこと

```
❌ 「製造オペレーションに強みがある」と書く
   → ✅ 「歩留まり率98%、業界平均95%。
         製造コスト比率32%、競合A 35%、競合B 38%」と書く

❌ 「R&D効率が高い」と書く
   → ✅ 「R&D売上比率5%、新製品売上比率30%。
         競合AはR&D比率7%で新製品比率25%」と書く

❌ 「バリューチェーン全体が最適化されている」と書く
   → ✅ 「調達から製造への情報連携システムあり。
         在庫回転率は過去5年で8回転→12回転に向上」と書く
```

### ✅ 心がけること

```
✅ すべての活動を数値で把握しようとする
✅ 競合との差を数値で示す
✅ 差がある場合、何がその差を生んでいるかの事実を探る
✅ 時系列での変化を追跡する
```
