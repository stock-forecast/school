# Forecast: Revenue v1.0

## プロンプトの目的

このプロンプトは、**統合レポートと成長分析の結果に基づき、セグメント別の5年間売上予想を策定**することを目的とする。

### 位置づけ

```
調査(5本) → 統合(2本) → 成長分析 → 【5年予想】 → debate
                                      ↑ここ

5年予想の役割：
- 予測期間（5年）の売上成長を具体化
- debateではこの5年目からRONIC∞/RR∞への収束を議論
```

### ⚠️ 最重要原則：根拠の明示

```
このプロンプトでやること：
✅ セグメント別に5年間の売上を予想する
✅ 各予想の根拠（事実・前提）を明示する
✅ 成長ドライバーを特定する
✅ 過去トレンドとの整合性を確認する

このプロンプトでやらないこと：
❌ 根拠なく「成長する」と予想しない
❌ 楽観・悲観シナリオは作成しない（中立のみ）
❌ 売上以外（利益、CF等）は予想しない
```

## 入力

1. **統合レポート**
   - integrate-industry.md 結果（業界構造、成長トレンド）
   - integrate-company.md 結果（成長経路、K拡大計画）

2. **成長分析レポート**
   - research-growth.md 結果（K, r, N, N/K）

3. **財務データ**
   - quantitative-data.json（過去の売上実績、セグメント別売上）

4. **会社公表情報**（任意）
   - 中期経営計画
   - セグメント別見通し

## 注意事項

### ❌ やってはいけないこと

```
❌ 根拠なく「年率10%成長」と予想する
   → ✅ 「市場成長5% + シェア拡大3% + 価格効果2% = 10%」
         と分解して根拠を示す

❌ 全セグメント同じ成長率を適用する
   → ✅ セグメントごとの成長特性を反映する

❌ 過去トレンドを無条件に延長する
   → ✅ 成長ステージの変化を考慮する

❌ 会社計画をそのまま採用する
   → ✅ 会社計画は参考とし、独自に予想する
```

### ✅ 心がけること

```
✅ 成長率を分解して説明する（市場成長、シェア、価格等）
✅ 成長率の推移（加速/減速）の理由を説明する
✅ セグメントの成長ステージを考慮する
✅ 整合性（過去トレンド、成長分析、会社計画）を確認する
✅ リスク要因（上振れ/下振れ）を整理する
✅ Debateへの接続（Y5以降の方向性）を意識する
```
