# Debate v2.0: 継続価値と永久成長率の判定

## プロンプトの目的

このプロンプトは、**収集・統合された事実に基づき、3つの視点からDebateを行い、永久成長率（g∞）を導出**することを目的とする。

### 継続価値の決定要因

```
企業価値の大部分は継続価値で決まる。
継続価値を決定づけるのは以下の3要素：

1. 不の解消 → RONIC∞の源泉（価値創造の原点）
2. K引き上げ → RR∞の源泉（成長余地の拡大）
3. 参入障壁構築 → RONIC∞の持続性（価値創造の防御）

これらが「企業DNA」として経営者交代後も継続するかが核心。
```

### ⚠️ 最重要原則：根拠のぶつかり合い

```
Debateの本質:
❌ 意見のぶつかり合い
   「参入障壁は高いと思う」vs「低いと思う」

✅ 根拠のぶつかり合い
   「事実Aがあるから高い」vs「事実Bがあるから低い」

3つの視点の違い:
- 同じ事実を見ている
- 事実の「重み付け」が異なる
- どの事実を重視するかで結論が変わる
```

## 永久成長率（g∞）推定の枠組み

### バリュードライバー式

```
永久成長率 g∞ = RONIC∞ × RR∞

RONIC∞ = 長期的に持続可能な新規投資資本利益率
RR∞ = 長期的に持続可能な再投資率
```

### 継続価値3要素とバリュードライバーの関係

```
┌─────────────────────────────────────────────────────────┐
│                    継続価値の決定                        │
└─────────────────────────────────────────────────────────┘
                          │
         ┌────────────────┼────────────────┐
         ▼                ▼                ▼
┌─────────────┐   ┌─────────────┐   ┌─────────────┐
│  不の解消   │   │  K引き上げ  │   │ 参入障壁構築 │
│（価値創造） │   │（成長余地） │   │ （持続性）  │
└──────┬──────┘   └──────┬──────┘   └──────┬──────┘
       │                 │                 │
       ▼                 ▼                 ▼
   RONIC∞の            RR∞の           RONIC∞の
    源泉               源泉             持続性
       │                 │                 │
       └────────┬────────┴────────┬────────┘
                ▼                 ▼
            RONIC∞              RR∞
                │                 │
                └────────┬────────┘
                         ▼
                    g∞ = RONIC∞ × RR∞
```

## 注意事項

### ❌ やってはいけないこと

```
❌ 入力レポートにない事実を追加する
   → すべての根拠は入力レポートから

❌ 3視点で異なる事実を参照する
   → 同じ事実を、重み付けを変えて解釈する

❌ 継続価値3要素を評価せずにg∞を推定する
   → 不の解消、K引き上げ、参入障壁構築を必ず評価する

❌ 企業DNAの継続性を無視する
   → 経営者交代後の継続性を必ず評価する

❌ RONIC∞を継続価値3要素と切り離して設定する
   → 不の解消(源泉) + 参入障壁(持続性)から導出する

❌ RR∞をK引き上げ能力と切り離して設定する
   → K拡大能力 + N/K から導出する
```

### ✅ 心がけること

```
✅ 継続価値3要素（不の解消、K引き上げ、参入障壁）を核心に据える
✅ 企業DNAが経営者交代後も継続するかを重視する
✅ g∞ = RONIC∞ × RR∞の分解を常に明示する
✅ 「仕組み」と「属人性」を区別する
✅ 争点を隠さず、正直に列挙する
✅ 判定の限界を明記する
```
