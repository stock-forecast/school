# 永久成長率分析ワークフロー v3.1

## 概要

このワークフローは、**バリュードライバー式に基づく永久成長率（g∞）の推定**を通じて、企業の継続価値を評価するためのプロンプト群である。

### バリュードライバー式

```
g∞ = RONIC∞ × RR∞

g∞     = 永久成長率
RONIC∞ = 長期的に持続可能な新規投資資本利益率
RR∞    = 長期的に持続可能な再投資率
```

### 継続価値の決定要因

```
企業価値の大部分は継続価値で決まる。
継続価値を決定づけるのは以下の3要素：

1. 不の解消 → RONIC∞の源泉（価値創造の原点）
2. K引き上げ → RR∞の源泉（成長余地の拡大）
3. 参入障壁構築 → RONIC∞の持続性（価値創造の防御）

これらが「企業DNA」として経営者交代後も継続するかが核心。
```

## 設計原則

### 1. 事実と評価の完全分離

```
調査・統合フェーズ: 事実収集のみ（評価しない）
判定フェーズ: 事実に基づく評価（根拠のぶつかり合い）
```

### 2. 継続価値3要素を核心に据える

```
1. 不の解消 → RONIC∞の源泉
2. K引き上げ → RR∞の源泉
3. 参入障壁構築 → RONIC∞の持続性

すべてが「企業DNA」として継続するかを評価
```

### 3. バリュードライバー式で一貫

```
g∞ = RONIC∞ × RR∞

- Tierはフェードレートではなく、RONIC∞のレベルを定義
- RR∞はN/KとK拡大能力から導出
- 財務実績との整合性を必ず確認
```

### 4. 3視点Debateによる判定

```
楽観・悲観・中立の3視点が同じ事実を見て、
重み付けを変えて議論する「根拠のぶつかり合い」
```

## 理論的背景

### Porter's Five Forces (1980)

競争構造の分析フレームワーク。

- 既存競合間の競争
- 新規参入の脅威
- 代替品の脅威
- 買い手の交渉力
- 売り手の交渉力

### Porter's Diamond Framework (1990)

国・地域の競争優位を決定する環境要因。

**内生要因（4つ）**
1. 要素条件（Factor Conditions）
2. 需要条件（Demand Conditions）
3. 関連・支援産業（Related and Supporting Industries）
4. 企業戦略・構造・競争（Firm Strategy, Structure, and Rivalry）

**外生要因（2つ）**
5. 政府（Government）
6. 偶発事象（Chance）

### Porter's Value Chain (1985)

企業の価値活動の分析フレームワーク。

**主活動**: 購買物流、製造、出荷物流、マーケティング・販売、サービス
**支援活動**: 全般管理、人事・労務管理、技術開発、調達活動

### 企業DNA（9要素MECE構造）

経営者が変わっても継続する価値創造の仕組み。

|  | Why（なぜ） | What（何を） | How（どう） |
|--|------------|-------------|------------|
| **ヒト** | 採用基準 | 組織設計 | 行動規範 |
| **モノ・カネ** | 投資基準 | 資源配分 | 資本規律 |
| **情報・知識** | 使命・理念 | 戦略思想 | 知識継承 |

### ロジスティック成長モデル

```
dN/dt = r × N × (1 - N/K)

N = 現在の規模
K = 環境収容力（市場容量）
r = 内的自然増加率（成長速度）
N/K = 浸透率（成長余地の逆数）
```

## Tier定義

| Tier | 定義 | RONIC∞ vs WACC | 継続価値3要素の状態 |
|------|------|----------------|-------------------|
| 1.0 | 永続的価値創造 | RONIC∞ >> WACC | 3要素すべて強固、DNA継続 |
| 1.5 | (中間) | - | - |
| 2.0 | 長期的価値創造（10年以上） | RONIC∞ > WACC | 3要素概ね良好、DNA継続 |
| 2.5 | (中間) | - | - |
| 3.0 | 中期的価値創造（5-10年） | RONIC∞ ≈ WACC | 一部要素に課題、DNA部分継続 |
| 3.5 | (中間) | - | - |
| 4.0 | 短期的価値創造（3-5年） | RONIC∞ < WACC | 複数要素に課題、DNA脆弱 |
| 4.5 | (中間) | - | - |
| 5.0 | 限定的価値創造（3年未満） | RONIC∞ << WACC | 3要素すべて課題、DNA不明確 |
| 5.5 | (中間) | - | - |
| 6.0 | 価値破壊 | RONIC∞ → 0 | 事業継続困難 |

## 補助プロンプト

メインワークフローとは独立して使用可能なプロンプト。

| ファイル | 目的 | 出力 |
|---------|------|------|
| `estimate-beta.md` | ファンダメンタルベータの推定 | β（WACC計算用） |

### estimate-beta.md の特徴

株価変動ではなく、**事業基盤の変動性（ROEボラティリティ）** からベータを推定する。

```
【Step 1】基準β算出（定量）
基準β = (対象企業のROEボラ / TOPIXのROEボラ) × 1.0

【Step 2】調整（定性）
調整後β = 基準β + 調整係数

調整要因（5つ）：
├─ オペレーティングレバレッジ（固定費比率）
├─ 財務レバレッジ（D/E比率）
├─ 事業の循環性（景気感応度）
├─ 収益の予測可能性（企業DNA、参入障壁）
└─ 収益源の分散度（セグメント・地域）
```

**ベンチマーク**：
- TOPIX β = 1.0
- J-REIT β = 0.7

## 使用方法

### Step 1: 調査フェーズ（並列実行可能）

```bash
# 5つの調査プロンプトを並列で実行
research-5-forces.md    → [企業名]_5forces.md
research-cluster.md     → [企業名]_cluster.md
research-value-chain.md → [企業名]_valuechain.md
research-corporate.md   → [企業名]_corporate.md
research-dna.md         → [企業名]_dna.md
```

### Step 2: 統合フェーズ

```bash
# 業界統合
integrate-industry.md
  入力: [企業名]_5forces.md + [企業名]_cluster.md
  出力: [企業名]_industry.md

# 企業統合
integrate-company.md
  入力: [企業名]_valuechain.md + [企業名]_corporate.md + [企業名]_dna.md
  出力: [企業名]_company.md
```

### Step 3: 成長フェーズ

```bash
research-growth.md
  出力: [企業名]_growth.md
```

### Step 4: 予想フェーズ

```bash
forecast-revenue.md
  入力: [企業名]_industry.md + [企業名]_company.md + [企業名]_growth.md
  出力: [企業名]_forecast.md
```

### Step 5: 判定フェーズ

```bash
debate.md
  入力: 全レポート（10本） + quantitative-data.json
  出力: [企業名]_debate.md
```

## 入力データ要件

### quantitative-data.json

判定フェーズで必要な財務データ。

```json
{
  "company": "企業名",
  "ticker": "証券コード",
  "fiscal_years": ["2020", "2021", "2022", "2023", "2024"],
  "roic": [8.5, 9.2, 10.1, 9.8, 10.5],
  "ronic": [12.0, 15.0, 14.0, 13.5, 14.2],
  "wacc": 7.5,
  "reinvestment_rate": [45, 50, 48, 52, 55],
  "organic_growth_rate": [5.4, 7.5, 6.7, 7.0, 7.8],
  "dividend_payout_ratio": [30, 28, 32, 30, 29]
}
```

**必須項目**:
- ROIC推移
- RONIC推移（増分NOPAT / 増分IC）
- WACC
- 再投資率推移（(NOPAT - FCF) / NOPAT）
- オーガニック成長率推移（全体成長率 - M&A寄与）
